M68000 Addressing Modes

The Motorola 68000 supports a wide range of addressing modes. Below is a categorized reference with syntax and examples.

1. Data Register Direct

D0 ... D7

Operand is in a data register.

Example: MOVE D0, D1 (copy contents of D0 into D1)

2. Address Register Direct

A0 ... A7

Operand is in an address register (cannot be used for byte-sized data).

Example: MOVEA.L D0, A1 (move longword from D0 into A1)

3. Address Register Indirect

(An)

The address register holds the effective address of the operand in memory.

Example: MOVE (A0), D0 (load value from memory at A0 into D0)

4. Address Register Indirect with Post-increment

(An)+

Use the address in An, then increment An (by operand size).

Example: MOVE (A0)+, D0 (copy value at A0 into D0, then increment A0)

5. Address Register Indirect with Pre-decrement

-(An)

Decrement An (by operand size), then use it as address.

Example: MOVE D0, -(A7) (push D0 onto stack using A7)

6. Address Register Indirect with Displacement

(d16,An)

Effective address = An + 16-bit signed displacement.

Example: MOVE (12,A0), D0 (copy from address A0+12 into D0)

7. Address Register Indirect with Index

(d8,An,Xn)

Effective address = An + sign-extended 8-bit displacement + index register (Xn = Dn or An, scaled).

Example: MOVE (4,A0,D1), D0 (copy from [A0 + 4 + D1] into D0)

8. Absolute Short

(xxx).W

16-bit absolute address (sign-extended).

Example: MOVE ($1234).W, D0

9. Absolute Long

(xxx).L

32-bit absolute address.

Example: MOVE ($00123456).L, D0

10. Program Counter with Displacement

(d16,PC)

Effective address = PC + 16-bit displacement.

Example: MOVE (12,PC), D0

11. Program Counter with Index

(d8,PC,Xn)

Effective address = PC + 8-bit displacement + index register.

Example: MOVE (4,PC,D1), D0

12. Immediate

#data

The operand is a constant value encoded in the instruction.

Example: MOVE #10, D0

💡 Tip: Address register indirect modes with pre-decrement/post-increment are especially useful for stack operations.
