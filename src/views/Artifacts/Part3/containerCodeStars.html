Now it’s time to add the Z axis to our demo coding. How do we use the Z axis and display it on screen? What we need to do is divide the X and Y axes by the Z axis, so it appears as if we have a 3D display.

Now I have a ready binary that contains 1000 random X, Y, Z values. So we must begin by reading all the X, Y, and Z values, three at a time. For example, we first get the initial X, Y, Z values from the random star data.

Here we need to use two new instructions: EXT and DIVS. EXT.L Dn takes the low word (D0–D7) and sign-extends it into the high word. DIVS is signed division (word divisor, longword dividend → word quotient, word remainder). DIVU is the unsigned version (it works the same, but with unsigned arithmetic).

Here we need to use DIVS, as there are also negative values in the random stars binary. 

			lea Stars,a0		;get address of stars bin
			move.w	(a0)+,d0	; Gte x value
			move.w	(a0)+,d1	; Get yvalue

			divs 	d2,d0		; Divide signed x by z

	muls	siny,d0		x * sin(y)

	asr.l	d6,d2

	move.w	d2,temp1	new z

	asr.l	d6,d0

	move.w	temp2,destx

	move.w	desty,d1

************************************

	muls	cosz,d0		x * cos(z)
	muls	sinz,d1		y * sin(z)
	sub.l	d1,d0		x * cos(z) - y * sin(z)
	asr.l	d6,d0
*	swap	d0
	move.w	d0,temp1

	move.w	destx,d0
	move.w	desty,d1
			 	
	muls	cosz,d1
	muls	sinz,d0
	add.l	d0,d1
	asr.l	d6,d1

	move.w	temp2,desty

	move.w	desty,d1
	move.w	destz,d2

	move.w	d1,(a3)+
	move.w	d2,(a3)+
