The LEA instruction loads the address of a source operand into an address register.

In this case, we load the hardware register $BFE001 into A0. Address registers range from A0 to A7, and are used for pointer-based memory access.

After this operation, A0 contains the address of $BFE001.

MOVE Instruction

The MOVE instruction transfers data from a source to a destination. Its size can vary depending on the operation:

  • MOVE.B — moves 8 bits (1 byte)
  • MOVE.W — moves 16 bits (1 word)
  • MOVE.L — moves 32 bits (1 longword)

When using parentheses, such as (A0), we are accessing the data stored at the memory address held in A0.

Data registers (D0–D7) are used for general-purpose values. After reading from (A0), the value is stored in D0.

Test and Branch Instructions

The TST instruction checks whether a value is zero or not. In this context, it is used to determine whether input data (such as a key press) is present.

BEQ (Branch if Equal) causes a jump when the result of a previous operation is zero.

In every demo, there is a main loop that runs once per frame and executes all pending routines.

A simpler—but less accurate—approach is to poll the vertical beam position directly. However, this is not an ideal solution, as we will see later.

Relying on beam polling can lead to timing inconsistencies compared to using proper vertical blank synchronization.

The CMP instruction compares a source value with a destination value and updates the condition flags based on the result. This is often followed by a branch instruction.

For example, BNE (Branch if Not Equal) will loop until the comparison condition is no longer met, commonly used here to wait for the vertical beam position.

As every demo coder knows, we must take control of the machine by switching away from the operating system. This involves disabling system interrupts and installing our own interrupt handlers.

The register responsible for this is INTENA at address $DFF09A. Bit 15 controls the global interrupt state: if it is 0, interrupts are disabled; if it is 1, interrupts are enabled.

Therefore, writing #$7FFF to INTENA disables all interrupts, while writing #$8FFF enables them. 

 
         move.w $dff01c,d5      ;read the status of interrupts in d5

In a simple setup, we define how many bitplanes we want to use for the color depth.

The number of bitplanes determines the number of available colors:

  • 1 bitplane = 2 colors
  • 2 bitplanes = 4 colors
  • 3 bitplanes = 8 colors
  • 4 bitplanes = 16 colors
  • 5 bitplanes = 32 colors

We must also define the actual color values for these bitplanes.

Colors are stored in registers from $DFF180 upward, depending on the number of bitplanes. These correspond to COLOR00 through COLOR31.

Setting the Number of Bitplanes

The number of bitplanes is controlled by the BPLCON0 register.

A commonly used formula is: 

ScreenDepth * $1000 + $0200 → BPLCON0

In Copper list format, this is typically written as: 

BPLCON0, ScreenDepth * $1000 + $0200

where ScreenDepth is the number of bitplanes used.

Bitplane Pointers

We also need to define bitplane pointer registers in order to access the Amiga’s 24-bit memory space.

These pointers are split into high and low words:

  • BPL1PTH / BPL1PTL — bitplane 1
  • BPL2PTH / BPL2PTL — bitplane 2
  • BPL3PTH / BPL3PTL — bitplane 3
  • BPL4PTH / BPL4PTL — bitplane 4
  • BPL5PTH / BPL5PTL — bitplane 5

These define where each bitplane is located in memory.

Display Window and DMA Settings

We must also configure display window size and screen memory width. Typical default values are: 

DIWSTRT  = $2AC1
DIWSTOP  = $2981
DDFSTRT  = $0038
DDFSTOP  = $00D0

The values $0038 and $00D0 define a standard 40-byte wide screen (320 pixels).

Modulo Registers

Two additional important registers are BPL1MOD and BPL2MOD.

These define the modulo (offset) between bitplane rows. In most basic setups, they are set to 0. However, they become necessary when using overscan or non-standard screen widths.

Now that we have the Copper list, how do we set the bitplane pointers for the picture? This is where we introduce a useful loop instruction: DBF / DBRA.

The DBF (Decrement and Branch if False) instruction decrements a counter and continues looping until it reaches -1. At that point, the loop terminates.

Screen Memory Calculation

A full-screen image has a width of 320 pixels, which equals 40 bytes per line (320 / 8).

With a height of 256 pixels, the total memory for one bitplane is: 

256 × 40 = 10,240 bytes per bitplane

For a 5-bitplane (32-color) display, we multiply by the number of bitplanes: 

10,240 × 5 = 51,200 bytes total

This gives the final memory size required for the full picture.

Setting Bitplane Pointers

Now it is time to assign the bitplane pointers to the image data in memory.

The SWAP instruction swaps the upper and lower halves of a register.

At this point, we have connected the Copper list bitplane pointers to the screen memory (the picture). However, we still need to define the colors in the Copper list.

Bitmap Memory Layouts

There are two common ways to store bitmaps: sequential and interleaved.

In a sequential bitmap, each bitplane is stored one after another in memory.

In an interleaved bitmap, the data is stored line by line, with all bitplanes grouped per scanline.

Example of a 3-bitplane interleaved layout: 

Line 0 Bitplane 0
Line 0 Bitplane 1
Line 0 Bitplane 2

Line 1 Bitplane 0
Line 1 Bitplane 1
Line 1 Bitplane 2
