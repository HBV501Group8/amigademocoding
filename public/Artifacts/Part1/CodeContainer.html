Waiting for a key click and signal no error 
 
lmb:        
        LEA     $BFEC01,A0      ; CIA-A serial data register
        MOVE.B  (A0),D0         ; read keycode (blocks until valid)
        TST.B   D0              ; check if nonzero
        BEQ     waitkey
        RTS                     ;close program
If we look up the LEA command, it fetches the address of the source and stores it in the destination. So we store the register $BFE001 in A0 (address registers can range from A0–A7). Now A0 contains the address of register $BFE001. The MOVE command, as its name implies, moves data from source to destination. The size of the MOVE instruction can be MOVE.B for moving a byte, MOVE.W for 16 bits, and finally MOVE.L for moving 32 bits. Moving A0 in parentheses like (A0) means accessing the data at the address stored in A0. D0 is one of the data registers (ranging from D0–D7). So the D0 register now contains the first byte from the address in A0. The TST instruction simply checks if D0 has data or not (whether a key has been pressed or not). BEQ stands for Branch if Equal (it branches when the result is zero).

As every demo coder knows, we will have to switch from the operating system and take control of the machine. First, we will disable the interrupts that the operating system is using and register our own interrupt.

So we need to begin by disabling all interrupts. The register that handles this is INTENA, or $DFF09A. If bit 15 is 0, interrupts are disabled; if it is 1, interrupts are enabled. Therefore, moving #$7FFF into INTENA disables all interrupts, just like moving #$8FFF into INTENA enables all interrupts. 

 
         move.w $dff01c,d5      ;read the status of interrupts in d5
         move.w #$7fff,INTENA   ; Clear all system interrupts

          btst	#6,$bfe001      ;Test left mouse button
          bne	lmb             ;Looped until pressed
          bset     #15,d5       ;turn on bit 15(enable)
          move.w  d5,INTENA     ; Restore system interrupts
          move.w  #0,d0         ;Signal no error to dos         
          rts                   ;close program

Now there is one more thing to cover, and that is DMA (Direct Memory Access). The register DMACON ($DFF096) controls which hardware components have direct memory access enabled. So we must do the same thing as with INTENA and COP1LCH ($DFF080): save the system’s current configuration and set up our own DMA settings. There is also a special read register, DMACONR ($DFF002), which we can use to save the current DMA state directly.

There are a few more relevant registers that we also have to set. These are used to define the number of bitplanes, preferably within the Copper list. We need bitplane pointers (high and low words) in order to access the 24-bit address space that the Amiga provides. These are the BPL pointers: BPL1PTH / BPL1PTL for one bitplane (2 colors) BPL2PTH / BPL2PTL for two bitplanes (4 colors) BPL3PTH / BPL3PTL for three bitplanes (8 colors) BPL4PTH / BPL4PTL for four bitplanes (16 colors) BPL5PTH / BPL5PTL for five bitplanes (32 colors)

We also have to set values for window size and screen memory width. The default values are: DIWSTOP $2981, DIWSTRT $2AC1, DDFSTRT $0038, DDFSTOP $00D0 The values $0038 and $00D0 are for the default 40-byte wide screen, or 320 pixels.

Now we have the Copper list, but how do we set those BPL pointers for the picture? That is simple. Now I introduce a special loop command called DBF or DBRA. If the counter reaches -1, the loop terminates. DBF stands for Decrement and Branch if False.

A fullscreen picture takes 320 / 8 = 40 bytes per line. A picture at full height is 256 pixels. So the screen memory is: 256 × 40 bytes = 10,240 bytes. Then we have to multiply by the screen depth, which is 5. Final outcome is 51200 bytes for picture

Now it's time to set the bitplane pointers to those in picture 

 

      #ScreenWidth = 320 / 8
      screenheight = 256
      screendepth = 5
      lea.l	BplPointers,a0	                ;Get bitplane pointers in copper
      move.w	#ScreenDepth-1,d1             ;counter
      move.l	Screen,d0                     ;Get screen adress
NextPlane
      move.w	d0,6(a0)	                ;And set low address word
      swap.w	d0		                ;Low = High : High = Low
      move.w	d0,2(a0)	                ;And set high adress word
      swap	d0
      add.l	#ScreenWidth*screenheight,d0 ; Next bitplane
      addq.w	#8,a0                          ;Advance to the next BplPointer in copper
      dbra	d1,NextPlane                   ; until -1

What SWAP does is swap between register halves. Now we have connected the Copper list BPL pointers to screen memory (the picture). But we will also have to set the colors in the Copper list.

There is one important thing to explain: there are two layouts for bitmaps—sequential and interleaved. In sequential bitmaps, each bitplane is stored one after another. In interleaved bitmaps, the data is stored line by line across all bitplanes. Here is an example of a 3-bitplane interleaved layout: 

        Line 0 Bitplane 0
        Line 0 Bitplane 1
        Line 0 Bitplane 2
        Line 1 Bitplane 0
        Line 1 Bitplane 1
        Line 1 Bitplane 2
