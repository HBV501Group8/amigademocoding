Amiga OCS / ECS — Custom Chips

Simple visual explanation of Agnus, Denise and Paula and how they connect the CPU to screen and sound.

At a glance

Agnus
Address generator — memory arbiter & copper
Denise
Video — bitplanes, sprites, playfields
Paula
Audio & I/O — 4 PCM channels, disk, ports

How data flows

  • CPU requests memory and instructs the chips.
  • Agnus controls access to chip RAM and runs the copper co-processor to change display timing/regs mid-frame.
  • Denise reads bitplane memory (provided by Agnus) and composes the final video output (bitplanes + sprites).
  • Paula reads audio samples and manages disk/serial I/O.

OCS vs ECS — short

OCS is the original set (Amiga 1000/500/2000). ECS adds higher-resolution modes and some improved timings (Amiga 500+, 3000).

Quick tips / interesting bits

  • Copper is a tiny co-processor inside Agnus that can change hardware registers on specific scanlines for effects like split-screen, gradients and synced sprites.
  • Bitplanes store the image in planar form — more planes = more colors, but the same horizontal resolution.
  • Sprites are hardware objects Denise can overlay with priority control — cheap and fast for games.

Copper

Runs during the beam's drawing and can change registers mid-frame — used for split screens and palette tricks.

Audio channels

Paula provides 4 independent PCM channels. Games mix effects and music using them directly.

OCS vs ECS

ECS adds an extra video mode and some timing improvements, but the architecture (Agnus/Denise/Paula) stays the same.
