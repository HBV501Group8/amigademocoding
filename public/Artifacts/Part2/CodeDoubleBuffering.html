Double buffering is a technique used to eliminate flickering and prevent partially drawn frames from being displayed on screen.

It works by using two bitplane buffers:

  • Buffer A: The currently displayed (front) buffer
  • Buffer B: The back buffer used for drawing

While one buffer is shown on the screen, the CPU or Blitter draws the next frame into the other buffer in the background.

Once rendering is complete, the buffers are swapped by updating the hardware bitplane pointers (for example, BPL1PTH/BPL1PTL, etc.).

This makes the newly drawn buffer visible instantly, while the previous screen becomes the new drawing target.

Using Double Buffering with the Blitter

In this example, we use the Blitter to draw the image into the back buffer, and then switch buffers to display the finished frame on screen.

This ensures smooth animation without visible tearing or partial updates.

Here we use the Blitter to display the image on the screen and also introduce double buffering.
