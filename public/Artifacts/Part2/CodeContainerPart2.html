🔹 What is the Blitter?

The Blitter (short for Block Image Transfer) is a dedicated hardware component inside the Amiga’s Agnus chip. Its main job is to move and manipulate blocks of memory—usually graphics data such as pixels—extremely quickly, without relying heavily on the CPU.

You can think of it as a hardware-accelerated copy-and-draw engine that operates in parallel with the Motorola 68000 CPU.

Using the Blitter

In this example, we use the Blitter to copy an image directly to the screen.

Here we use the blitter to put the image to screen

NextPlane:

            swap.w	d0		Low = High : High = Low

 ; Save system copper

copyimage
	lea	image2,a0           ;Get the address of picture in a9
	move.l	#screen,a1      ;Get the adddress of the screen in a1
WBlit	btst	#14,$dff002		Blitter done
	bne.s	WBlit			If not wait

	move.l	a0,bltapth      ; source is a0 or the picture
	move.l	a1,bltdpth      ; Destination is the screen
	move.l	#-1,bltafwm     ; Usual -1 for the mask    
	move.w	#0,bltamod      ; No modulo for source
	move.w	#0,bltdmod      ; No modulo for destination
	move.l	#$09f00000,bltcon0  ; simple copy op
	move.w	#(64*42*3)+20,bltsize   ; this means blit the whole Logo
                                    ; Because we are using INTERLEAVED bitmaps
                                    ; by using this formula 42 is height of Logo
                                    ; 3 is the number of Biplanes and 20 is 40 bytes
                                    ; represnted in word. So we are using the whole screen
                                    ; horizontally
	rts



CopperList:
