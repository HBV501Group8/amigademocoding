How can we fill a 3D cube with different colors on each face? The Blitter chip makes this possible by enabling fill mode. However, to use the Blitter for filling, we must rely on a specific line-drawing routine that correctly sets the fill carry bit. This ensures the fill operation works properly after drawing the cube’s edges.

What Does the Blitter Do?

A blitter is a hardware circuit designed to quickly move (“blit”) blocks of data—typically pixels or bitplanes—between memory locations. It can also apply logical operations such as AND, OR, and XOR during the transfer.

In the Amiga system, the blitter is especially powerful and supports:

  • Line drawing (similar to the Bresenham algorithm)
  • Area filling
  • Fast image copying and masking
  • Logical operations using three source channels (A, B, C) and one destination (D)

While the blitter cannot directly draw arbitrary polygons, it provides a fill mode that works after the polygon’s outline has been drawn.

How Filling Works

The general process is:

  • Draw the edges of the polygon using the blitter’s line mode
  • Enable fill mode to automatically fill the interior pixels

The fill operation relies on a mechanism called fill carry. As the blitter scans horizontally across each line:

  • Encountering an edge pixel toggles the fill state
  • When fill carry is active, pixels are filled
  • The next edge toggles filling off

This follows the classic even–odd rule: enter the polygon → fill → exit → stop.

Enabling Fill Mode

To activate fill mode, specific bits in the BLTCON1 register must be set:

  • Bit 4 (FILL MODE): Set to 1 to enable filling
  • Bit 2 (Fill Carry):
    • 0 → start unfilled
    • 1 → start filled

Depending on the hardware revision, additional control bits (such as line mode flags) may also need to be configured.

Here is an example 

FillPage:
	btst	#6,$dff002
	bne.s	Fillpage        Wait for blitter
	move.w	#4-1,d7         nr of bitplanes
	move.l	wscr,a0         Screen address
	add.l	#40*256,a0      negin for bottom to top
	add.l	#40*255,a0
	add.l	#40*255,a0
	add.l	#40*255,a0
fillloop
	btst	#6,$dff002
	bne.s	Fillloop
	move.w	#$09f0,$dff040	; BLTCON0
	move.w	#$0012,$dff042		; BLTCON1 (EFE!DESC)
	move.l	#$ffffffff,$dff044	; BLTAFWM
	move.l	a0,$dff050		; BLTAPT
	move.l	a0,$dff054		; BLTDPT
	move.l	#0,$dff064		; BLTAMOD&BLTDMOD
	move.w	#(255*64)+20,$dff058
	sub.l	#40*256,a0
	dbra	d7,fillloop

	rts
