How can we fill a 3D cube with different colors on each face? The Blitter chip has this capability by enabling fill mode. However, to use the Blitter for filling, we must use a specific line-drawing routine that sets the fill carry bit correctly. This routine is designed to work with the Blitter's fill mode, allowing us to fill the interior of the cube's faces after drawing the edges.

How can we set the Blitter to fill mode? Well, let’s remember the role of the Blitter.

A blitter is a hardware circuit that quickly moves (or “blits”) blocks of data (usually pixels or bitplanes) from one part of memory to another, often applying logical or arithmetic operations (e.g., AND, OR, XOR). In the Amiga’s case (the most famous blitter), it can perform: Line drawing (Bresenham-like) Area filling Fast image copying or masking Logical combinations of three source channels (A, B, and C) and one destination (D)>

The Amiga blitter didn’t directly draw arbitrary polygons — instead, it provided fill mode that you could use after you’ve drawn the polygon’s outline (edges). The idea was: Draw the edges of the polygon (using the blitter’s line draw mode). Then, use the blitter fill mode to automatically fill the inside pixels between edges.

The fill logic uses a concept called "fill carry" — a bit that flips every time the blitter encounters an edge pixel while scanning a line horizontally. Think of it like this: The blitter scans across a line of pixels. When it sees an edge pixel (set bit), it toggles its fill state. If “fill carry” is on, pixels get filled until the next edge toggles it off. This produces the standard “even–odd” fill rule (enter polygon → fill → exit → stop).

To Enable Fill Mode

You set bit 0 of BLTCON1 (BLTCON1_F_LINE) and bit 4 of BLTCON1 (BLTCON1_F_FILL) depending on hardware revision. Typical configuration (Amiga 500–1200 style): Set BLTCON1 bit 4 (FILL MODE) to 1 → this activates the fill logic. The fill logic state (fill carry) is in bit 2 of BLTCON1: 0 → clear fill carry (start unfilled) 1 → set fill carry (start filled)

FillPage:
	btst	#6,$dff002
	bne.s	Fillpage        Wait for blitter
	move.w	#4-1,d7         nr of bitplanes
	move.l	wscr,a0         Screen address
	add.l	#40*256,a0      negin for bottom to top
	add.l	#40*255,a0
	add.l	#40*255,a0
	add.l	#40*255,a0
fillloop
	btst	#6,$dff002
	bne.s	Fillloop
	move.w	#$09f0,$dff040	; BLTCON0
	move.w	#$0012,$dff042		; BLTCON1 (EFE!DESC)
	move.l	#$ffffffff,$dff044	; BLTAFWM
	move.l	a0,$dff050		; BLTAPT
	move.l	a0,$dff054		; BLTDPT
	move.l	#0,$dff064		; BLTAMOD&BLTDMOD
	move.w	#(255*64)+20,$dff058
	sub.l	#40*256,a0
	dbra	d7,fillloop

	rts
